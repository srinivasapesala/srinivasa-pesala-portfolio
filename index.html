SriniMart Data Agent — Architecture

Self-Service Analytics for 300-Store Retail

Five-Layer Architecture
L1

Agent Runtime

LangGraph state graph orchestrates the full query lifecycle — receives natural language, routes to the right reasoning pattern, maintains conversation state across turns, and coordinates all five layers to produce a governed answer.

LangGraph State Graph Conversation Memory Reasoning Router
L2

Tool Layer

50+ SriniMart data sources wrapped as MCP servers, loaded on-demand (500-token startup cost vs. 15,000 upfront). Tool Search meta-tool dynamically discovers the right 3–5 tools per query. Programmatic batch calling handles multi-store operations in a single code block rather than 300 LLM round-trips.

MCP Protocol Tool Search Batch Calling Few-Shot Examples
L3

Data & Context (RAG)

Hybrid dense + keyword retrieval over SriniMart's schema catalog and business glossary. Maps plain-English terms to exact column names — "revenue" → sales.total_amount, "Q3" → July–September — so the agent uses real columns instead of hallucinating plausible-sounding ones.

FAISS Azure Cognitive Search Business Glossary Schema Embeddings
L4

Guardrails

Dual-check access control: pre-query WHERE clause injection scopes SQL to the user's permitted stores/region; post-query answer validation is the safety net. PII masking runs on every answer before it surfaces. Per-role rate limiting enforces query budgets. Store managers see only their store. Regional managers see only their region.

RBAC PII Masking Rate Limiting Scope Injection
L5

Evaluation

Every query is recorded with latency, token cost, reasoning pattern used, and PII-leak flag. Per-pattern cost analytics reveal if queries are being misrouted to expensive patterns. Targets: latency P95 < 5s, average cost < $0.05/query, error rate < 5%, PII leakage 0%.

Latency P95 Token Cost LangSmith Prometheus
Reasoning Pattern Routing
ReAct ~80% of queries

Think → Act → Observe loop for simple data lookups and single-metric questions. Low cost, low latency — the default for most stakeholder queries.

Plan-and-Execute Multi-step reports

Planner creates a full step-by-step plan; Analyst executes each step sequentially. Used for quarterly reviews and cross-region comparisons where upfront coordination pays off.

Reflexion Root-cause queries

Execute → Reflect on what failed → Retry with insight. Converts each dead end into a better-targeted next hypothesis. Used for "why did Southwest revenue drop?" investigations.

Tree-of-Thought Strategic decisions

Generates three parallel reasoning branches, evaluates each, pursues the best. Reserved for high-value strategic questions where quality justifies the 3–5x cost multiplier.
